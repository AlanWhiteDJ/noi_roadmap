## 📖 故事引入

小明记性不好。每次要用一个数字，他都重新输一遍。

第一天：算 3 + 5 → 他在纸上写 3 和 5，算出 8

第二天：用昨天的 8 继续算 → 他又得重新输一遍 3 + 5

第三天……他烦了，心想："我就不能把 8 存起来吗？给它贴个标签，什么时候要用就拿过来？"

这就是**变量**的由来。变量 = 内存里的一个贴了标签的格子。你把数字放进去，给它取个名字。以后要用这个数字，叫它名字就行。

## 🧱 知识讲解

### 4.1  变量是什么

把计算机的内存想象成一排排储物柜。每个柜子可以放一个数字。

**声明一个变量** = 你找一个空柜子，贴上一张标签，规定这个柜子里只能放什么类型的东西。

**给变量赋值** = 你把东西放进这个柜子。

**使用变量** = 你按标签找到柜子，取出里面的东西。

```cpp
int score;      // 声明：找一个柜子，标签写 score，只放 int
score = 95;     // 赋值：把 95 放进 score 的柜子
cout << score;  // 使用：打开 score 柜子，取出里面的数
```

### 4.2  声明变量

声明的格式很简单：

```
类型  变量名;
```

例如：

```cpp
int age;              // 声明一个装整数的变量，叫 age
double height;        // 声明一个装小数的变量，叫 height
long long bigNumber;  // 声明一个装大整数的变量，叫 bigNumber
```

可以一行声明多个同类型的变量：

```cpp
int a, b, c;  // 声明三个 int 变量
```

也可以声明的同时赋值（这叫**初始化**）：

```cpp
int score = 100;       // 声明并初始化为 100
double pi = 3.14159;   // 声明并初始化为 3.14159
```

### 4.3  变量命名规则

C++ 给变量命名有严格的规定，也有约定俗成的习惯。规定是铁律，习惯是修养。

**硬性规定（违反会报错）**：

1. 只能由字母（A-Z, a-z）、数字（0-9）、下划线（_）组成
2. 第一个字符不能是数字
3. 不能是 C++ 的关键字（比如 `int`、`return`、`if`）

**合法 vs 非法示例**：

| 变量名 | 合法？ | 原因 |
|--------|--------|------|
| `score` | ✅ | 字母组成 |
| `my_score` | ✅ | 字母+下划线 |
| `player2` | ✅ | 字母开头，含数字 |
| `2player` | ❌ | 数字不能开头 |
| `my-score` | ❌ | 不能有减号 |
| `int` | ❌ | 关键字 |
| `我的分数` | ❌ | 不能有中文（某些编译器允许但不推荐） |

**推荐习惯（不遵守不会报错，但别人会皱眉）**：

1. 变量名要有意义。`score` 比 `s` 好，`studentCount` 比 `x` 好
2. 用英文。哪怕英文不好，也比拼音强——`length` > `changdu`
3. 多个单词用下划线连接（`max_score`）或驼峰命名（`maxScore`），任选一种、保持一致

> 💡 一个好变量名 = 不用看上下文就知道它是干什么的。

### 4.4  赋值

`=` 在 C++ 里不是"等于"，而是**"把右边的值放进左边的变量"**。

```cpp
int x;
x = 10;       // 把 10 放进 x
x = 20;       // 把 20 放进 x（原来的 10 被覆盖了）
x = x + 5;    // 把 x 当前的值（20）加上 5，结果（25）存回 x
```

最后一行需要仔细理解：**先算右边的表达式，再把结果赋给左边**。所以 `x = x + 5` 不是数学上的"x 等于 x+5"（那是不可能的），而是"把 x 加上 5，再存回 x"。

### 4.5  未初始化的危险

**声明变量时不赋值，变量里是什么？**

```cpp
int x;
cout << x;  // 输出什么？可能是 0，可能是 32767，可能是一个内存垃圾值
```

答案是：**不确定**。取决于那块内存之前被谁用过、留下了什么垃圾数据。

> ⚠️ 未初始化变量是 C++ 中最常见的 bug 来源之一。变量声明时，**养成同时赋初始值的习惯**。

如果你确实不知道初始值该设多少，可以设 0：

```cpp
int sum = 0;       // 累加器，从 0 开始
int maxVal = -1;   // 最大值标记
```

### 4.6  声明和赋值的不同写法

以下四种写法效果完全一样，都是声明一个 int 变量 score 并设为 100：

```cpp
// 写法一：声明和赋值分开
int score;
score = 100;

// 写法二：声明同时初始化（推荐）
int score = 100;

// 写法三：用括号初始化
int score(100);

// 写法四：用大括号初始化（C++11 风格）
int score{100};
```

推荐写法二——最简单、最清晰。

### 4.7  常量和 const

有些值在整个程序运行期间都不该变。比如圆周率 π = 3.14159。如果你不小心改动了它，后果很严重。

C++ 用 `const` 关键字来标记"这个变量不可更改"：

```cpp
const double PI = 3.1415926;
PI = 3.14;  // ❌ 编译错误！常量不能修改
```

竞赛中，常量用在几个经典场景：
- 数学常数（PI）
- 数组最大大小（`const int MAXN = 100005;`）
- 模数（`const int MOD = 1000000007;`）

## ⚠️ 容易犯的错

### 错 1：把 = 当等于

❌ 思维惯性：`x = x + 1` 在数学上不可能
✅ C++ 现实：把 x 当前值加 1，存回 x

### 错 2：不初始化直接用

❌ `int sum; cout << sum;` // 输出垃圾值
✅ `int sum = 0; cout << sum;` // 输出 0

### 错 3：变量名无意义

❌ `int a, b, c, d;` // 天知道它们代表什么
✅ `int math, english, chinese, total;` // 一看就懂

### 错 4：拼写不一致

❌ 声明 `int studentScore;`，使用 `cout << studentscore;` // 大小写不同
✅ C++ 区分大小写，`studentScore` ≠ `studentscore`

```

```

| 维度 | 目标 |
|------|------|
| **知识** | 区分六种基本类型（int/long long/double/char/string/bool）；理解溢出；知道 sizeof |

## 🗺️ 教学流程

### 第 1 课时：整数类型与溢出（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | 讲小刚的 85 亿分故事。"你觉得问题出在哪？" |
| 5-15 分 | 讲授 | 数据类型 = 杯子。int = 中杯（约 21 亿），long long = 大杯（约 9×10¹⁸） |
| 15-25 分 | 演示 | **溢出实验**：`int a = 2147483647; a = a + 1; cout << a;` 观察从最大值翻到最小值 |
| 25-35 分 | 实操 | 学生声明 int 和 long long 变量，算一些大的乘积 |
| 35-45 分 | 口诀 | "普通数字用 int，超过 20 亿换 long long" |

### 第 2 课时：double / char / string / bool（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 复习 | 快速回顾 int 和 long long |
| 5-15 分 | 讲授 | double：小数专用。整数除法 vs 浮点除法对比 |
| 15-25 分 | 讲授 | **char vs string 对比**——这是本章最容易混淆的点。板书两边对照：左边 char 单引号 'A'，右边 string 双引号 "ABC"。口诀："单引号一个，双引号一串" |
| 25-35 分 | 讲授 | bool：只有 true 和 false。演示输出 bool 结果（显示 1/0），让学生猜为什么 |
| 35-45 分 | 互动 | 快速判断题："全班人数"用什么类型？"你的名字"？"是否迟到"？——抢答 |

### 第 3 课时：sizeof + 实战（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 讲授 | sizeof 用法。板书条形图对比各类型大小 |
| 10-35 分 | 实操 | 动手试试 1-4。特别是试试 4（输出 bool），观察 true→1、false→0 |
| 35-45 分 | 分类练习 | 给出一系列场景（人数、面积、姓名首字母、是否及格…），学生写正确的声明语句 |

### 第 4 课时：纠错 + 练习（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-15 分 | 纠错 | "容易犯的错"四条。重点放在错 3（char 和 string 搞混）——让学生故意写出错误代码并观察编译器报错 |
| 15-40 分 | 练习 | 学生做练习题 1-4 + 挑战题 5（分类连线） |
| 40-45 分 | 讲评 | 重点讲提高题第 3 题（整数除法陷阱）和挑战题 5 |

### 第 5 课时：小测 + 小结（45 分钟）

标准流程：思维导图（六种类型三组分类）→ 小测 → 互批 → 展望。

## 🔥 重点难点

### 重点

1. **int vs long long 的选择**："乘法之前先想范围"
2. **溢出**：必须亲眼看到、亲手触发
3. **char（单引号）vs string（双引号）**：本章最容易混淆的点
4. **bool 的存在感**：为 M2 的条件判断做铺垫

### 卡点突破

| 卡点 | 突破 |
|------|------|
| char vs string 混淆 | 板书对照 + 口诀。让学生尝试 `char c = "AB";` 看报错 |
| bool 输出 1/0 不理解 | 解释：C++ 内部 true=1, false=0，这只是显示方式 |
| string 为什么需要 `#include <string>` | 类比：不同工具放不同柜子。iostream 柜子里有 cin/cout，string 柜子里有 string 类型 |
| 六种类型记不住 | 三组分类法：数字（int/long long/double）、文字（char/string）、判断（bool） |

## 💬 常见问答

**Q1："为什么 bool 输出是 1 和 0 不是 'true' 和 'false'？"**

A：C++ 内部把 true 存为 1，false 存为 0。cout 直接输出的话就显示 1 和 0。如果你想让屏幕上显示 "true"，可以用 `cout << boolalpha << b;`。（不必现在就记住，知道有这个事就行。）

**Q2："char 和 string 能互相转吗？"**

A：可以，但现阶段不纠结。先记住：一个字用 char，一串字用 string。等学了更多操作再回头研究转换。

**Q3："int 不够用的时候，能不能全部用 long long？"**

A：可以，很多竞赛选手就是这么干的。但养成"全用 long long"的习惯也有坏处——数组很大时多占一倍内存。所以知道什么时候该用什么，比一刀切更好。

## 🎲 课堂活动

### 活动 1：类型连连看（第 2 课时用）

**道具**：白板分两栏。左栏贴场景（全班人数 / 圆周率 / 姓名首字母 / 是否下雨 / 你的名字 / 全球人口），右栏贴类型名（int / long long / double / char / string / bool）

**规则**：学生依次上来连线。连线后要说出理由。其他学生评判对错。

### 活动 2：溢出接力赛（第 1 课时用）

**规则**：分两组。老师出算式（如 100000 × 100000），每组抢答"int 能装下吗？如果不能，用什么？"。答对记 1 分，答错扣 1 分。

## ✅ 习题详解

### 基础题

**1. 选择题**
（1）B（4 字节）  
（2）B（-1 亿在 int 范围内）  
（3）C（"Hello" 是一串字符 → string）  
（4）B（单字符 'A' → char）  
（5）D（是否及格 = 真/假 → bool）

**2. 填空题**
（1）8  
（2）溢出  
（3）单、双  
（4）true、false

### 提高题

**3. 分析题**
```
a = 2（5/2 整数除法）
b = 2.0（5/2 整数除法得 2，赋值给 double 变成 2.0）
c = 2.5（5.0 触发浮点除法）
输出：2 2 2.5
```

**4. 编程题**
20! ≈ 2.43×10¹⁸，在 long long 范围内。用 int 会在中途溢出。

### 挑战题

**5. 分类题**
```
全班人数（45）         → int
圆周率（3.14159）      → double
你的名字              → string
考试等级（A/B/C/D）    → char
今天是否下雨           → bool
```

**6. 探究题**
int 最小值 = -2,147,483,648，最大值 = 2,147,483,647。负数多一个是因为补码表示中 0 占了一个正数位置。

## 🗺️ 教学流程

### 第 1 课时：变量是什么（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "小明的烦恼"故事——每次用数字都要重新输，烦死了 |
| 5-15 分 | 讲授 | 变量 = 贴了标签的储物柜。类比：声明 = 找柜子贴标签，赋值 = 放东西进去，使用 = 打开柜子取出 |
| 15-25 分 | 板书 | 在黑板画一排"储物柜"，标签上写变量名。把数字放进去 |
| 25-40 分 | 实操 | 学生声明 int、double、long long 变量，赋值，输出。每个学生至少声明 5 个不同变量 |
| 40-45 分 | 对比 | 声明并初始化 vs 声明后赋值——两种方式都展示 |

### 第 2 课时：赋值和命名（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 复习 | 快速回顾变量声明 |
| 10-20 分 | 讲授 | `=` 不是等于，是"把右边的放进左边"。重点讲 `x = x + 1`，在黑板上一步步演示执行过程 |
| 20-35 分 | 讲授 | 命名规则。硬性规定 + 推荐习惯。在黑板上列出合法 vs 非法示例。让学生判断"哪些名字能用" |
| 35-45 分 | 讨论 | 展示一段变量名混乱的代码（`int a, b, c, d, e;`），让学生猜它在干什么。再展示命名良好的版本——体会差距 |

### 第 3 课时：危险与保护（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 演示 | **恐怖时刻**：声明 int x; 不初始化，cout x。反复运行 3 次，每次结果可能不同。解释：内存里可能有"垃圾" |
| 10-20 分 | 讲授 | 初始化的重要性。"变量就像冰箱里的保鲜盒——你不知道之前放过什么，先洗干净（初始化为 0）再用" |
| 20-30 分 | 讲授 | const 常量。什么时候用：已知不变的值（PI、数组大小） |
| 30-45 分 | 实操 | 动手试试 1-3 |

### 第 4-5 课时：练习 + 小测（90 分钟）

标准流程。

## 🔥 重点难点

### 重点

1. **`=` 是赋值不是等于**：这是编程入门的核心认知转折。必须反复强调
2. **未初始化变量**：让学生亲眼看到"垃圾值"，比讲一百遍"要初始化"都管用
3. **命名有意义**：这是整个职业生涯的好习惯

### 卡点突破

| 卡点 | 突破 |
|------|------|
| `x = x + 1` 理解不了 | 一步步追踪：①看 x 当前值（比如 5）②算 5+1=6 ③把 6 放回 x。用动画式板书呈现 |
| 声明和赋值混淆 | 用两个动词区分：`int x;` = 搭建（柜子），`x = 5;` = 放入（物品） |
| 命名总用拼音 | 不强求英文好。给一个"常用变量名中英对照表"贴在机房墙上 |

**常用变量名对照表**（建议贴在机房）：

| 含义 | 推荐命名 |
|------|----------|
| 总和 | sum / total |
| 计数 | cnt / count |
| 最大值 | maxVal / mx |
| 最小值 | minVal / mn |
| 答案 | ans / result |
| 临时变量 | tmp / temp |
| 数组大小 | n / m / MAXN |
| 循环变量 | i / j / k |
| 标记/旗帜 | flag / vis |

## 💬 常见问答

**Q1："为什么 C++ 不允许变量名用中文？"**

A：历史上编程语言都是英文世界的产物。虽然某些编译器的确支持中文变量名（`int 分数 = 90;` 能编译通过），但强烈不推荐——换了编译器可能就不认了，而且国际比赛、团队协作不可能用中文。

**Q2："初始化为 0 就够了？还是有什么讲究？"**

A：看用途。累加器初始化为 0，累乘器初始化为 1，最大值初始化为一个很小的数（如 -1e9），最小值初始化为一个很大的数（如 1e9）。不知道初始化什么的时候就设 0。

**Q3："const 和普通变量到底有什么区别？编译器不让我改而已，我为什么不能用普通变量？"**

A：可以，但 const 有两个好处：①代码意图清晰（"这个值不应改变"），别人读你的代码秒懂；②编译器会帮你检查——手滑改了 const 的值马上报错，用普通变量就悄悄错了。这是防御性编程。

### 活动：变量命名大挑战

**规则**：
- 老师给出一个变量的"用途描述"
- 学生在 10 秒内写出一个合法的变量名
- 写得最有意义且合法的人记 1 分
- 例如："存一个班级的人数"→ `classSize` / `studentCount` / `n`

**陷阱题**：
- "存第二个学生的分数"（不能 `2score`——数字不能开头！）
- "存 return 的值"（不能叫 `return`——关键字！）

## ✅ 习题详解

### 基础题

**1. 选择题**
（1）C（`_count` 合法）  
（2）B（10，赋新值覆盖旧值）  
（3）B（3×2=6）

**2. 填空题**
（1）`long long distance = 0;`  
（2）`const double G = 9.8;`  
（3）数字

### 提高题

**3. 分析题**

```
a=5, b=5（b 得到 a 的副本）
a=8（修改 a，不影响 b）
b=10（8+2）
输出：8 10
```

关键：`b = a` 是**值拷贝**，之后 a 和 b 各自独立。

**4. 编程题（参考答案）**

```cpp
int chinese = 90, math = 95, english = 88;
int total = chinese + math + english;
double avg = total / 3.0;  // 注意 3.0 不是 3
cout << "总分：" << total << " 平均分：" << avg << endl;
```

### 挑战题

**5. 交换题**

```cpp
int a = 10, b = 20;
int temp = a;  // 第一步：把 a 暂存到 temp
a = b;         // 第二步：把 b 放进 a
b = temp;      // 第三步：把暂存的 a 放进 b
// 结果：a=20, b=10
```

类比：A 杯橙汁、B 杯苹果汁。需要第三个空杯 C。A→C，B→A，C→B。

## 📦 补充题库

1. 以下变量名哪些合法？score、_count、2nd、my-var、myVar、class、MAX_VALUE  
（合法：score、_count、myVar、MAX_VALUE；非法：2nd 数字开头、my-var 有减号、class 是关键字）

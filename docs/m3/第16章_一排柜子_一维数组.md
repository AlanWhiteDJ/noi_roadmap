## 📖 故事引入

### 班主任的噩梦

开学第一周，班主任王老师拿到全班 50 个学生的期末成绩。她想把成绩输入电脑算平均分，于是她写：

```cpp
int score1, score2, score3, score4, score5;
int score6, score7, score8, score9, score10;
// ……写到 score50 的时候，王老师已经不认识键盘了
```

更崩溃的是——算平均分：

```cpp
double avg = (score1 + score2 + score3 + ... + score50) / 50.0;
```

这一行代码比清明上河图还长。而且如果隔壁班有 55 个人，所有代码都得重写。

王老师崩溃了。隔壁教信息学的李老师微微一笑："你用数组啊——`int scores[50]`，一行搞定。"

```cpp
int scores[50];  // 50 个柜子，整齐排好
for (int i = 0; i < 50; i++) {
    cin >> scores[i];        // 循环输入，50 行变 3 行
}
```

### 一排储物柜

小区楼下有一排储物柜，共 20 个，编号 0 到 19。每个柜子大小一样——只能放同样尺寸的包裹。

你去取快递。短信说"您的快递在 7 号柜"。你走到第 7 个柜子前，打开，拿到包裹。你不会去开 8 号柜——那不是你的。

这就是数组！**数组 = 一排编号从 0 开始的储物柜，每个柜子装同一种类型的东西**。

## 🧱 知识讲解

### 16.1  什么是数组

数组（array）是一组**相同类型**的变量，它们在内存里**一个挨一个排好**，用一个共同的名字来管理，用**编号（下标）**来区分。

> 🔑 三个关键词：**相同类型、连续存储、下标访问**。

类比：储物柜 = 数组，柜子编号 = 下标，柜子里的东西 = 数组元素。

### 16.2  声明数组

声明数组要告诉计算机三件事：
1. 柜子里装什么类型（`int`、`double`、`char`……）
2. 柜子的名字
3. 有多少个柜子（用方括号 `[]` 指定数量）

```cpp
int scores[50];         // 50 个整数柜子，装成绩
double temps[7];        // 7 个小数值子，装一周温度
char grades[30];        // 30 个字符柜子，装等级
```

> 声明时，方括号里的数字必须是**常数**（编译时就能确定的值）。不能是变量！

> ❌ **错误写法——先看到坑，再学怎么绕：**
>
> ```cpp
> int n;
> cin >> n;
> int arr[n];  // ❌ 变量 n 不是常数！部分编译器可能通过，但不符合 C++ 标准
> ```
>
> ✅ **正确写法**——先用题目给定的最大范围声明：
>
> ```cpp
> int arr[1005];  // 题目说 n ≤ 1000，我声明 1005，留缓冲
> int n;
> cin >> n;
> for (int i = 0; i < n; i++) cin >> arr[i];
> ```
>
> 等到学了 `vector`（下册），你就可以用变量大小了。

### 16.3  下标——柜子的编号

数组用**下标**（index）来访问每一个元素。**C++ 中下标从 0 开始**。

```cpp
int arr[5];              // 5 个柜子
arr[0] = 10;             // 第 1 个柜子放 10
arr[1] = 20;             // 第 2 个柜子放 20
arr[2] = 30;
arr[3] = 40;
arr[4] = 50;             // 第 5 个柜子（最后一个！）
```

| 下标 | 0 | 1 | 2 | 3 | 4 |
|------|---|---|---|---|---|
| 值 | 10 | 20 | 30 | 40 | 50 |
| 第几个 | 第 1 个 | 第 2 个 | 第 3 个 | 第 4 个 | 第 5 个 |

> 🧠 **记住这个公式**：一个有 n 个元素的数组，下标范围是 `0` 到 `n-1`。不存在下标为 n 的元素！

> ❌ **常见踩坑——下标从 1 开始：**
>
> ```cpp
> int arr[5];
> for (int i = 1; i <= 5; i++) {
>     cin >> arr[i];  // ❌ arr[5] 越界！只有 arr[0]~arr[4]
> }
> ```
>
> ✅ **正确写法：**
>
> ```cpp
> for (int i = 0; i < 5; i++) {
>     cin >> arr[i];  // ✅ i 从 0 到 4，安全
> }
> ```

### 16.4  初始化数组

你可以在声明的同时"装货"——这叫**初始化**：

```cpp
int arr[5] = {10, 20, 30, 40, 50};  // 5 个全装好
int evens[6] = {2, 4, 6};           // 前 3 个装好，后 3 个自动填 0
int zeros[100] = {0};               // 全部初始化为 0（常用技巧！）
int prices[] = {15, 25, 35, 45};    // 编译器自动数：4 个元素
```

> 💡 `int zeros[100] = {0};` 是竞赛里最常用的初始化方式——100 个柜子全部变成 0。

> ❌ **常见踩坑——没初始化就读：**
>
> ```cpp
> int arr[10];
> cout << arr[3];  // ❌ 输出什么？不知道！可能是 0，也可能是奇怪的垃圾值
> ```
>
> ✅ **正确：** 要么初始化 `int arr[10] = {0};`，要么先赋值再读。

### 16.5  用 for 循环遍历数组

数组的天然搭档是 `for` 循环。因为下标是连续整数（0, 1, 2, 3, …），正好用循环变量 i 依次访问：

```cpp
int scores[5] = {95, 88, 76, 92, 80};

// 输出全部成绩
for (int i = 0; i < 5; i++) {
    cout << "第" << i + 1 << "个同学：" << scores[i] << "分" << endl;
}
```

输出：

```
第1个同学：95分
第2个同学：88分
第3个同学：76分
第4个同学：92分
第5个同学：80分
```

> 🎯 循环变量 `i` 从 0 到 4（不是 1 到 5！），因为数组下标从 0 开始。

常见遍历模式：

```cpp
// 输入：逐个读入数组元素
for (int i = 0; i < n; i++) {
    cin >> arr[i];
}

// 求和
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += arr[i];
}

// 找最大值
int maxVal = arr[0];
for (int i = 1; i < n; i++) {
    if (arr[i] > maxVal) maxVal = arr[i];
}
```

> ❌ **常见踩坑——循环条件写成 `<=`：**
>
> ```cpp
> int arr[5] = {10, 20, 30, 40, 50};
> for (int i = 0; i <= 5; i++) {  // ❌ i 会变成 5，访问 arr[5] 越界！
>     cout << arr[i] << " ";
> }
> ```
>
> ✅ **正确：**
>
> ```cpp
> for (int i = 0; i < 5; i++) {   // ✅ < 不是 <=，i 最多到 4
>     cout << arr[i] << " ";
> }
> ```
>
> 🧠 口诀：**n 个元素，i 从 0 到 n-1，循环条件是 `i < n`。**

### 16.6  小综合：计算平均成绩

```cpp
#include <iostream>
using namespace std;

int main() {
    int scores[5] = {95, 88, 76, 92, 80};
    int sum = 0;

    for (int i = 0; i < 5; i++) {
        sum += scores[i];
    }

    double avg = (double)sum / 5;  // 强制转 double，保证小数不丢
    cout << "总分：" << sum << endl;
    cout << "平均分：" << avg << endl;

    return 0;
}
```

输出：

```
总分：431
平均分：86.2
```

### 16.7  数组越界——绝对不能碰的红线

假如你声明了 `int arr[5];`（下标范围 0~4），然后写了：

```cpp
arr[5] = 100;   // ⚠ 越界！没有 5 号柜子！
```

这会发生什么？答案是——**不知道**。

C++ 不会检查你是否越界。它可能：
- 静悄悄地改掉了别的变量的值（最可怕！）
- 程序崩溃
- 看起来一切正常（但埋下了定时炸弹）

> ⚠️ 数组越界是 C++ 新手最容易犯的、后果最无法预测的错误。**永远保证你的下标在 `[0, n-1]` 范围内。**

一个安全的习惯：定义数组时多给几个位置：

```cpp
int arr[105];  // 题目说最多 100 个，我声明 105 个，留缓冲
```

### 16.8  数组应用举例

**场景 1：存储一周温度**

```cpp
double temps[7] = {23.5, 25.0, 22.8, 26.1, 24.3, 27.0, 25.6};
for (int i = 0; i < 7; i++) {
    cout << "星期" << i + 1 << "：" << temps[i] << "℃" << endl;
}
```

**场景 2：反转数组**

```cpp
int arr[5] = {1, 2, 3, 4, 5};
// 前后交换：0↔4, 1↔3
for (int i = 0; i < 5 / 2; i++) {
    int temp = arr[i];
    arr[i] = arr[4 - i];
    arr[4 - i] = temp;
}
// 输出：5 4 3 2 1
```

## 🦶 你踩过这些坑吗？

- [ ] **坑 1**：把下标当第几个——`arr[1]` 是第 1 个？错，它是第 2 个（下标从 0 开始）
- [ ] **坑 2**：循环条件写 `i <= n`——结果访问了不存在的 `arr[n]`，越界
- [ ] **坑 3**：声明数组时用变量做大小——`int arr[n]` 不符合 C++ 标准

- [ ] **坑 6**：忘记"多留几个位置"——题目数据范围一变大，程序就崩

## 🧠 本章小结

```
数组 = 一排同类型的储物柜

声明：
  int arr[10];                // 10 个 int 柜子
  double temps[7] = {23.5};   // 7 个 double，第一个 23.5

下标：
  从 0 开始 → arr[0], arr[1], ..., arr[n-1]

遍历（黄金搭档）：
  for (int i = 0; i < n; i++) {
      操作 arr[i];
  }

初始化 = {0} → 全部清零（竞赛高频用法）

越界 → 访问 arr[n] 或更大 → 程序爆炸/静默错误 → 绝不！
```

## 📖 故事引入

放学后你走到自动售货机前，想买瓶饮料。

机器屏幕上显示菜单：1-可乐、2-雪碧、3-芬达、4-矿泉水、5-果汁……

你按了"3"，机器"咕咚"一声掉出一瓶芬达。

这个过程的代码逻辑大概是：

```
如果 选了 1 → 出可乐
如果 选了 2 → 出雪碧
如果 选了 3 → 出芬达
...
```

用 `else if` 写也能搞定，但你要是看代码——密密麻麻一排 `else if`，像一堵墙。而且每次判断都要把"选了哪个数字"重新算一遍，不够利索。

有没有一种写法，直接告诉程序："看这个数字是多少，跳到对应入口"？

有，这就是 `switch-case`。

## 🧱 知识讲解

### 10.1  switch-case 基本语法

`switch-case` 就像一个"跳转表"——你给一个整数（或字符），程序直接跳到对应的 `case` 标签开始执行：

```cpp
switch (表达式) {
    case 值1:
        // 表达式 == 值1 时执行这里
        break;
    case 值2:
        // 表达式 == 值2 时执行这里
        break;
    default:
        // 所有 case 都不匹配时执行这里
}
```

自动售货机的例子：

```cpp
int choice;
cin >> choice;

switch (choice) {
    case 1:
        cout << "你选了：可乐" << endl;
        break;
    case 2:
        cout << "你选了：雪碧" << endl;
        break;
    case 3:
        cout << "你选了：芬达" << endl;
        break;
    case 4:
        cout << "你选了：矿泉水" << endl;
        break;
    case 5:
        cout << "你选了：果汁" << endl;
        break;
    default:
        cout << "没有这个编号，请重新选择" << endl;
}
```

对比 `else if` 版本，`switch-case` 的好处一目了然：结构清晰，一眼就能看出有多少个选项，每个选项做什么。

### 10.2  break 的作用：制止"穿透"

**❌ 先看看这段代码——有什么问题？**

```cpp
int n = 2;
switch (n) {
    case 1: cout << "A";
    case 2: cout << "B";
    case 3: cout << "C";
}
// 输出：BC  ← 嗯？怎么 B 和 C 都出来了？
```

输入 2，期望只输出 "B"，结果 "BC" 都出来了！这是因为 `switch-case` 有个特别的设计——**case 标签只是"入口"，不是"出口"**。程序从匹配的 case 冲进去，然后就一路向下狂奔，把后面的 case 全执行一遍，直到遇到 `break` 或 switch 结束。

这叫做**穿透**——不加 `break`，程序就像瀑布一样从当前 case 一路流到底。

**✅ 正确版本：每个 case 末尾加 break**

```cpp
int n = 2;
switch (n) {
    case 1:
        cout << "A" << endl;
        break;
    case 2:
        cout << "B" << endl;    // 输出 B
        break;                  // 然后跳出，不往下掉
    case 3:
        cout << "C" << endl;
        break;
}
// 输出：B  ← 这才是我们要的
```

> 💡 **穿透是 bug 的温床**。除非你刻意利用穿透（下一节会讲），否则每个 case 末尾都加 `break`。

### 10.3  巧用穿透：合并相同处理

有时候多个 case 做同样的事，利用穿透可以写得非常漂亮：

```cpp
int month;
cin >> month;

switch (month) {
    case 1: case 3: case 5: case 7: case 8: case 10: case 12:
        cout << "31天" << endl;
        break;
    case 4: case 6: case 9: case 11:
        cout << "30天" << endl;
        break;
    case 2:
        cout << "28天或29天" << endl;
        break;
    default:
        cout << "无效月份" << endl;
}
```

看！1、3、5、7、8、10、12 月都是 31 天，写在一排 case 标签里，共享同一段代码。这比 `if (month==1 || month==3 || ...)` 清爽多了。

另一个常用场景——成绩等级简化版：

```cpp
int score;
cin >> score;

switch (score / 10) {   // 除以 10 取整：95→9，82→8，67→6
    case 10:            // 100 分
    case 9:
        cout << "A" << endl;
        break;
    case 8:
        cout << "B" << endl;
        break;
    case 7:
        cout << "C" << endl;
        break;
    case 6:
        cout << "D" << endl;
        break;
    default:
        cout << "E" << endl;
}
```

> 💡 这里 `score / 10` 是整数除法，能把分数映射到 0–10 的区间，再用 switch 一锤定音。这是 switch 的经典用法。

### 10.4  switch-case 的限制

switch 不是万能的，它有明确的限制：

1. **switch 后面的表达式必须是整型或字符型**（`int`、`long long`、`char`，以及后面会学的枚举）。不能是 `double`、不能是 `string`。

2. **case 后面必须是常量**——不能是变量，不能是范围（你不能写 `case 60..100:`）。

3. **各 case 的值不能重复**——编译器直接报错。

这些限制意味着：`switch-case` 适合"精确匹配某个值"的场景。如果你需要判断"分数 > 90"，那还是用 `if-else`。

### 10.5  三目运算符 `? :`：最简单的二选一

有时候你要做的判断非常简单——就一行赋值：

```cpp
// if-else 写法
int result;
if (a > b) {
    result = a;
} else {
    result = b;
}
```

天哪，写了 5 行就为了取个最大值。C++ 有个更简洁的写法——三目运算符 `? :`：

```cpp
int result = (a > b) ? a : b;   // 一行搞定！
```

格式：

```
条件 ? 成立时的值 : 不成立时的值
```

翻译成人话：**"条件成立吗？成立就选冒号前面的，不成立就选冒号后面的。"**

更多例子：

```cpp
int absValue = (x >= 0) ? x : -x;                    // 求绝对值
string status = (score >= 60) ? "及格" : "不及格";   // 输出文字
cout << ((n % 2 == 0) ? "偶数" : "奇数") << endl;     // 直接输出
```

> ⚠️ 三目运算符不要嵌套！`a ? b : c ? d : e` 这种写法只有写的人自己能看懂（过三天自己也不一定看得懂）。简单二选一用它，复杂的请老老实实写 `if-else`。

### 10.6  switch-case vs if-else：怎么选

| 场景 | 推荐 | 原因 |
|------|------|------|
| 判断范围（>  <） | `if-else` | switch 不支持范围 |
| 精确匹配几个值 | `switch-case` | 结构更清晰 |
| 条件涉及逻辑运算 | `if-else` | switch 里不能写 `&&` |
| 选项很多（5+） | `switch-case` | 代码更整洁 |
| 简单二选一赋值 | 三目 `? :` | 一行搞定 |

没有绝对的优劣，选让代码**更清晰好读**的那个。

## ⚠️ 你踩过这些坑吗

- [ ] **坑 1：忘了写 break**——程序从匹配的 case 一路执行到底，把后面的 case 代码也跑了。每个 case 末尾加 `break`，除非你故意的。
- [ ] **坑 2：用 switch 判断范围**——`case 60..100:` 不合法，范围判断用 `if-else`。
- [ ] **坑 3：switch 里用 string**——`switch (name) { case "小明": ... }` C++ 不支持，字符串比较用 `if-else`。
- [ ] **坑 4：三目运算符写得太复杂**——`int r = a > b ? (c > d ? e : f) : (g > h ? i : j);` 谁能看懂？三目只做简单二选一，复杂的回到 `if-else`。

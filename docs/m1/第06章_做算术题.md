## 📖 故事引入

小华在写一个"分糖果"程序。有 17 颗糖，要平均分给 5 个小朋友。

"17 ÷ 5 = 3.4，每人 3.4 颗？"小华犯难了——糖果不能掰成 0.4 颗啊。

他需要的其实是两个答案：
1. 每人能分到几颗？（整数除法）
2. 还剩几颗？（求余数）

这两个问题，C++ 各有一个运算符能直接解决。而且不只是糖果——取余数这个操作在编程里比你想象的常用得多。

## 🧱 知识讲解

### 6.1  算术运算符一览

C++ 有 7 种算术运算符：

| 运算符 | 名称 | 例子 | 结果 |
|--------|------|------|------|
| `+` | 加 | `5 + 3` | `8` |
| `-` | 减 | `5 - 3` | `2` |
| `*` | 乘 | `5 * 3` | `15` |
| `/` | 除 | `5 / 2` | `2`（整数除法） |
| `%` | 取模（余数） | `5 % 2` | `1` |
| `++` | 自增 | `a++` 或 `++a` | a 加 1 |
| `--` | 自减 | `a--` 或 `--a` | a 减 1 |

### 6.2  整数除法 vs 小数除法

这是本章最重要的一点：

```cpp
int a = 5 / 2;      // a = 2（小数部分丢弃，不是四舍五入！）
double b = 5 / 2;   // b = 2.0（先做整数除法得 2，再转成 double）
double c = 5.0 / 2; // c = 2.5（5.0 是 double，所以做小数除法）
```

规则很简单：**两个操作数都是整数 → 整数除法；任何一个操作数是浮点数 → 浮点除法。**

```cpp
// ❌ 错误理解：以为声明 double 就万事大吉
double b = 5 / 2;   // b = 2.0！先做整数除法得 2，再转成 double

// ✅ 正确写法：至少有一个操作数是浮点数
double c = 5.0 / 2; // c = 2.5（5.0 是 double，所以做小数除法）
```

这个规则会跟你好几年。请把它刻进脑子里。

### 6.3  取模运算 `%`

`%` 是"求余数"。17 ÷ 5 = 3 余 2，所以 `17 % 5 = 2`。

取模的经典用法：

**判断奇偶**：
一个数除以 2，余数如果是 0，它就是偶数；余数如果是 1，它就是奇数。等后面学了条件判断语句，你就可以让程序自动告诉你一个数是奇数还是偶数了。

**循环到头后回到起点**：
```cpp
int day = (today + 100) % 7;  // 今天星期几，100 天后星期几？
```

**提取个位数字**：
```cpp
int ge = n % 10;  // 123 % 10 = 3
```

> ⚠️ `%` 只能用于整数，不能用于浮点数。

### 6.4  自增和自减

这是一个让初学者困惑的地方：

```cpp
int a = 5;
int b = a++;   // b = 5, a = 6  （先赋 b，a 再加 1）
int c = ++a;   // c = 7, a = 7  （a 先加 1，再赋给 c）
```

| 写法 | 含义 | 执行顺序 |
|------|------|----------|
| `a++` | 后置自增 | 先用 a 的值，用完再加 1 |
| `++a` | 前置自增 | 先加 1，再用新的值 |
| `a--` | 后置自减 | 同 a++，但是减 1 |
| `--a` | 前置自减 | 同 ++a，但是减 1 |

**最佳实践**：如果只是单独让变量加 1（`a++` 或 `++a` 独占一行），选哪种都一样。初学者建议就用 `a++`，等熟练了再玩花样。

### 6.5  复合赋值运算符

这些是"运算 + 赋值"的简写：

| 写法 | 等价于 |
|------|--------|
| `a += 5` | `a = a + 5` |
| `a -= 5` | `a = a - 5` |
| `a *= 5` | `a = a * 5` |
| `a /= 5` | `a = a / 5` |
| `a %= 5` | `a = a % 5` |

```cpp
int score = 90;
score += 5;   // score = 95
score *= 2;   // score = 190
score /= 10;  // score = 19
```

不仅简洁，而且在复杂表达式中更不容易出错（变量名只出现一次）。

### 6.6  运算符优先级

如果你写 `3 + 5 * 2`，C++ 会先算 `5 * 2` 再算 `3 + 10`——和数学课一样，乘除优先级高于加减。

完整的优先级从高到低：

| 优先级 | 运算符 |
|--------|--------|
| 最高 | `()` 括号 |
| ↑ | `++` `--`（前置） |
| | `*` `/` `%` |
| | `+` `-` |
| 最低 | `=` `+=` `-=` 等赋值 |

**记不住全部优先级怎么办？——加括号。**

```cpp
// 不确定优先级的时候，用括号明确意图：
int result = (a + b) * (c - d);
```

> 💡 括号不仅让程序正确，也让读你代码的人轻松。额外多敲两个括号，比猜优先级划算一万倍。

### 6.7  整数运算的范围陷阱

```cpp
int a = 100000;
int b = 100000;
int c = a * b;  // 100 亿！int 装不下，溢出成垃圾值
cout << c;      // 输出奇怪的东西
```

**一旦乘法可能超出 `int` 范围，立即用 `long long`**：

```cpp
long long a = 100000;
long long b = 100000;
long long c = a * b;  // 正确：10 亿，在 long long 范围内
```

> 常用的安全范围速记：两个 10 万相乘 = 100 亿，必超 int。两个 46340 以下的安全。

## ⚠️ 你踩过这些坑吗

- [ ] **坑 1：整数除法误以为会得到小数** — `double x = 5 / 2;` 得到 2.0，不是 2.5。用 `double x = 5.0 / 2;`。
- [ ] **坑 2：取模用于浮点数** — `double x = 5.5 % 2;` 编译错误。取模只能用整数。
- [ ] **坑 3：用 / 取数位时忘了整数特性** — 想取 123 的十位：`123 / 10` = 12（整数除法），`12 % 10` = 2。正确！但别写成 `123 / 10.0`。
- [ ] **坑 4：复合赋值误以为是数学等式** — `x *= y + 1` 等价于 `x = x * (y + 1)`，不是 `x = x * y + 1`。

### 基础题

**1. 计算题**

（1）`17 / 4` = \_\_\_\_

```cpp
int a = 10;
```

**3. 选择题**

（1）`7 / 2` 的结果是？
&emsp; A. 3.5 &emsp; B. 3 &emsp; C. 4 &emsp; D. 3.0

&emsp; A. x=5, y=5 &emsp; B. x=6, y=5 &emsp; C. x=6, y=6 &emsp; D. x=5, y=6

（3）`a %= 3` 等价于？
&emsp; A. `a = a % 3` &emsp; B. `a = 3 % a` &emsp; C. `a = a / 3` &emsp; D. `a = 3 / a`

**4. 填空题**

（1）`int a = 5; a += 3; a *= 2;` 执行后 a = \_\_\_\_。
（2）`int n = 123; int ge = n % 10;` 执行后 ge = \_\_\_\_。
（3）判断一个整数 n 是偶数的条件是 `n % 2 == \_\_\_\_`。

### 提高题

**5. 编程题 — 三位数拆解**

输入一个三位整数（如 368），分别输出它的百位、十位、个位。

示例运行：

```
请输入一个三位数：368
百位：3
十位：6
个位：8
```

**6. 编程题 — 时间计算**

输入一个总分钟数（如 135），转换成"X 小时 Y 分钟"的格式。

**7. 分析题**

以下代码分别输出什么？先推算，再上机验证。

```cpp
int a = 10, b = 3;
cout << a / b << " " << a % b << endl;
cout << 10.0 / 3 << endl;

a = 5; b = a++;
cout << a << " " << b << endl;

a = 5; b = ++a;
cout << a << " " << b << endl;
```

### 挑战题

**8. 编程题 — 数字反转**

输入一个三位整数，输出它反转后的数字（如输入 123，输出 321）。

（提示：先取出个位、十位、百位，再重新组合。）

**9. 编程题 — 平均速度**

小明骑自行车去学校。去的时候上坡，速度 12 km/h；回来下坡，速度 20 km/h。路程来回各 6 公里。求全程的平均速度（保留 2 位小数）。

> 注意：平均速度 ≠ (12 + 20) / 2。想想为什么？需要总路程除以总时间。

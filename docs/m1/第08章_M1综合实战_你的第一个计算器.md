## 🧱 知识讲解

### 8.1  程序的骨架：IPO 模型

所有程序——不管多复杂——本质上都遵循一个三步骤模式：

```
输入（Input） → 处理（Process） → 输出（Output）
```

你用过计算器 APP 吧？你按数字和运算符（输入），它内部运算（处理），屏幕上显示出结果（输出）。这就是 IPO。

我们写计算器，也按这个骨架搭：

```
I：cin 接收两个数 + 一个运算符
P：根据运算符，做对应的计算
O：cout 输出结果
```

先把骨架画清楚，再往里面填肉。这比一上来就埋头敲代码高效十倍。

### 8.2  用选择结构判断运算符

用户可能输入 `+`、`-`、`*`、`/` 四种运算符中的一种。怎么让程序"知道"该做哪种运算？

```cpp
if (op == '+') {
    result = a + b;
} else if (op == '-') {
    result = a - b;
} else if (op == '*') {
    result = a * b;
} else if (op == '/') {
    result = a / b;
} else {
    cout << "不支持的运算符！" << endl;
    return 1;   // 异常退出
}
```

注意运算符用 `char` 类型存储——一个字符变量，刚好装一个 `'+'` 或 `'*'`。

### 8.3  守护除法：除数为 0 怎么办？

这是计算器里**最容易被攻击的 bug**。15 ÷ 0 在数学上没有意义，在 C++ 里会导致程序崩溃（浮点除法得到 `inf`，整数除法直接炸）。

写除法之前，加一堵"防火墙"：

```cpp
if (op == '/') {
    if (b == 0) {
        cout << "错误：除数不能为 0！" << endl;
    } else {
        result = a / b;
    }
}
```

这叫做**防御性编程**——假设用户会犯所有可能的错，提前挡住。养成这个习惯，你的代码质量就能甩开 80% 的初学者。

### 8.4  完整代码：迷你计算器

先看一个"典型初学者版本"——你能找出几个问题？

```cpp
// ❌ 错误版：隐患重重
#include <iostream>
using namespace std;

int main() {
    int a, b, result;    // 用 int → 不支持小数
    char op;
    cin >> a >> op >> b;
    if (op == '+') result = a + b;
    if (op == '-') result = a - b;
    if (op == '*') result = a * b;
    if (op == '/') result = a / b;   // 没检查除数为 0！
    cout << result << endl;
    return 0;
}
```

> 问题清单：① int 不支持小数 ② 除法没检查除数 ③ 没有处理非法运算符 ④ 用多个 if 而不是 else if（效率低）

✅ 把前面的零件组装起来——正确的版本：

```cpp
#include <iostream>
using namespace std;

int main() {
    double a, b, result;   // 用 double 支持小数
    char op;               // 用 char 存运算符

    cout << "=== 迷你计算器 ===" << endl;
    cout << "请输入算式（如 5 + 3）：" << endl;
    cin >> a >> op >> b;   // 一次读三个：数 运算符 数

    // 根据运算符选择计算方式
    if (op == '+') {
        result = a + b;
    } else if (op == '-') {
        result = a - b;
    } else if (op == '*') {
        result = a * b;
    } else if (op == '/') {
        if (b == 0) {
            cout << "错误：除数不能为 0！" << endl;
            return 1;
        }
        result = a / b;
    } else {
        cout << "错误：不支持的运算符 " << op << endl;
        return 1;
    }

    cout << a << " " << op << " " << b << " = " << result << endl;
    return 0;
}
```

> 💡 `cin >> a >> op >> b;` 一行读三个值——程序会智能地跳过空格，把数字读给 `a`，把字符读给 `op`，把下一段数字读给 `b`。这就是 C++ 输入流的便利之处。

### 8.5  好代码的三个习惯

写完程序不是终点，**"写得好"才是**。从你的第一个完整程序开始，养成三个习惯：

**① 有意义的变量名**

❌ `double a, b, c;` — 一个月后自己都看不懂
✅ `double num1, num2, result;` — 一眼就知道是干嘛的

**② 适当的注释**

不是每行都写注释，而是在关键位置写：

```cpp
// 除法需检查除数不为 0
if (b == 0) { ... }
```

**③ 一致的缩进**

每一层 `{}` 缩进 2 或 4 格，让代码像目录树一样层次分明。大多数 IDE 按一下 Tab 键自动缩进——养成敲 `{` 就顺手缩进的习惯。

## ⚠️ 你踩过这些坑吗

- [ ] **坑 1：整数除法导致结果错误** — 用 `int` 存操作数，输入 `7 / 2` → 输出 `3`。优先用 `double`，除非你确定用户只需要整数。
- [ ] **坑 2：`op` 用 `string` 存** — `string op;` 然后比较 `if (op == "+")`——能跑，但小题大做。单个字符用 `char`。
- [ ] **坑 3：忘了检查除数** — `result = a / b;` 当 `b = 0` 时程序崩溃。除法前先 `if (b == 0)` 检查。
- [ ] **坑 4：`cin >> a >> op >> b` 输入时空格不对** — 用户输入 `5+3`（没空格）也能正确解析，但建议加空格更清晰。

### 基础题

**1. 填空题**：IPO 模型中，I 代表 \_\_\_\_，P 代表 \_\_\_\_，O 代表 \_\_\_\_。

**2. 选择题**：用户输入 `8 / 0`，如果程序没有检查除数，会发生什么？
&emsp; A. 自动输出 0 &emsp; B. 程序崩溃或输出异常值 &emsp; C. 输出 8 &emsp; D. 自动退出

**3. 读代码**：以下程序输入 `10 + 5` 后输出什么？

```cpp
int a, b;
char op;
cin >> a >> op >> b;
if (op == '+') cout << a + b;
else if (op == '-') cout << a - b;
```

**4. 判断题**

（1）程序里的除数为 0 只会让结果变成 0，不会崩溃。（ ）
（2）`char` 是存单个字符的类型，适合存运算符。（ ）
（3）IPO 模型只适用于计算器程序。（ ）
（4）`cin >> a >> op >> b;` 可以一次性读取数字和字符。（ ）

### 提高题

**5. 编程题 — 升级计算器**

在迷你计算器的基础上增加"取模运算"（`%`）。用户输入 `17 % 5`，程序输出 `2`。注意：取模运算也需检查除数是否为 0，且操作数必须是整数（提示：用 `long long` 存）。

**6. 编程题 — 算式智能识别**

用户输入一个完整的算式字符串，如 `"15+23"`（不带空格），程序自动解析出两个操作数和运算符并计算结果。提示：遍历字符串找到运算符位置，再用 `substr` 拆分。

```cpp
int main() {
    int a, b; char op;

```
